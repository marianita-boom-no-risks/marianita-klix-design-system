Three steps. That's it.

No complicated setup. No config headaches.

1

Download the project

Grab the design system. Clone it with git, or just download the ZIP. 

git clone https://github.com/marianita-boom-no-risks/marianita-klix-design-system.git
or Download ZIP
2

Run the install script

One command. It installs MCP dependencies, symlinks the Claude skill into ~/.claude/skills/, and prints the JSON snippet to paste into your AI client. 

cd marianita-klix-design-system && ./install.sh

What this does: installs Node packages for the MCP server, sets up the Claude skill globally, and tells you exactly what to add to your AI client's config. Other flags available: --check (status), --mcp (deps only), --skill (link only), --uninstall.

Prefer to set up manually? See IDE-specific instructions below.
3

Start building!

Open your AI tool and try one of these prompts. The AI will automatically use the design system to generate clean, consistent code.

"Build me a settings page with toggles and a save button"

"Create a login page with social auth buttons"

"Design a dashboard with stat cards and a data table"

"Make a pricing page with three plan cards"

The differentiator

An MCP is good. An MCP plus a skill is better.

Most design-system MCPs just expose tools. Claude has to figure out the right order to call them, what rules to enforce, and when to push back on bad requests. This project includes a Claude skill that does all of that.

MCP alone

  • Tools are there, but Claude guesses what to call when
  • Easy to skip get_css_setup and produce broken output
  • No enforcement of the prefix / token / a11y rules
  • If you ask for a carousel, it'll invent one
Included

MCP + Claude skill

  • Tool order is encoded in the skill (no guessing)
  • 8 hard rules enforced automatically (no hardcoded values, mk- prefix, semantic HTML, WCAG AA, etc.)
  • Output contract: every reply has manifest, code, gaps, next steps
  • If you ask for something not in the registry, it offers real options instead of fabricating

What's inside the skill

A router that loads context on demand based on what you're asking for.

SKILL.md

Router. Hard rules. Workflow diagram. Token cheatsheet. Loaded every conversation.

references/state-machines.md

10 components with full state machines: CSS classes, ARIA attributes, keyboard maps.

references/a11y-playbook.md

WCAG 2.1 AA per-component patterns. Focus management, live regions, keyboard maps.

references/figma-workflow.md

Figma URL → MK-DS code, with OKLCH perceptual color matching, gap surfacing.

references/dark-mode-engineering.md

Token genealogy. Contrast tables for light + dark. Common bugs and their fixes.

references/motion.md

Duration / easing tokens. Reduced-motion patterns. Performance checklist.

references/anti-patterns.md

30+ failure modes from real PR reviews. Symptom → why it's wrong → fix.

references/migration.md

Bootstrap / Tailwind → MK-DS. Class-by-class mapping, JS migration, state preservation.

references/recipes.md

10 composition recipes: auth form, dashboard, settings, modal, table, pricing, more.

examples/01-login-page.md

Worked example: full transcript from request to final code, with self-audit.

examples/04-rejected-request.md

How the skill says "no" well. Real options instead of fabrication. Honest gaps.

scripts/audit-output.sh

Bash validator. 8 rules. Cross-checks against the component registry. Zero deps.

All free, all MIT, all inside the repo. .claude/skills/marianita-klix-ds/

Manual IDE setup

If you prefer to configure things yourself instead of using install.sh, expand your IDE below.

First install dependencies, then add to .claude/settings.json in your project: 

cd mcp && npm install
{
  "mcpServers": {
    "marianita-klix-ds": {
      "command": "node",
      "args": ["/path/to/marianita-klix-design-system/mcp/index.js"]
    }
  }
}

Replace /path/to with the actual path on your machine.

Install dependencies, then add to .cursor/mcp.json in your project: 

cd mcp && npm install
{
  "mcpServers": {
    "marianita-klix-ds": {
      "command": "node",
      "args": ["/path/to/marianita-klix-design-system/mcp/index.js"]
    }
  }
}

Replace /path/to with the actual path on your machine.

Install dependencies, then add to your Windsurf MCP settings: 

cd mcp && npm install
{
  "mcpServers": {
    "marianita-klix-ds": {
      "command": "node",
      "args": ["/path/to/marianita-klix-design-system/mcp/index.js"]
    }
  }
}

Replace /path/to with the actual path on your machine.

For any MCP-compatible tool, use the standard stdio transport configuration: 

cd mcp && npm install
{
  "mcpServers": {
    "marianita-klix-ds": {
      "transport": "stdio",
      "command": "node",
      "args": ["/path/to/marianita-klix-design-system/mcp/index.js"]
    }
  }
}

The server communicates over stdin/stdout using JSON-RPC. Any MCP client will work.
